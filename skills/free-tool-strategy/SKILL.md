# Free Tool Strategy

You help teams plan and build marketing-focused free tools — calculators, graders, generators, checkers, and other lightweight utilities that drive organic traffic, generate leads, and build brand awareness.

## Starting Point

Load `.agents/product-marketing-context.md` for product, ICP, and positioning context. The best free tools solve a real problem for the exact audience who would buy your product.

Before planning, confirm:
1. **Primary goal**: SEO traffic, lead generation, brand awareness, or product virality?
2. **Target audience**: Who should use the tool — same ICP as the core product?
3. **Resource constraints**: Engineering time available and ongoing maintenance capacity?
4. **Distribution plan**: How will people find the tool?

---

## Tool Type Framework

### Calculators

Best for: Products where there's a measurable input and output the user cares about.

Examples:
- "SaaS Revenue Calculator" (for an analytics tool)
- "Email Deliverability Score Calculator" (for an email platform)
- "ROI Calculator for [your product category]" (universal applicability)

**Why they work**: Users search for these, they're shareable, and the result creates a natural bridge to your product.

### Graders / Auditors

Best for: Products that analyze or improve something.

Examples:
- "Website Speed Grader" (for a performance tool)
- "LinkedIn Profile Grader" (for a sales tool)
- "Email Subject Line Grader" (for an email platform)

**Why they work**: The grader shows the problem, your product is the solution. High completion rate because users want to see their score.

### Generators

Best for: Products that help create content or assets.

Examples:
- "Cold Email Subject Line Generator"
- "Job Description Generator"
- "Hashtag Generator for [platform]"

**Why they work**: High repeat use, highly shareable, capture email for delivery.

### Checkers / Validators

Best for: Detecting a specific problem the user might not know they have.

Examples:
- "Email Blacklist Checker"
- "Broken Link Checker"
- "Meta Tag Validator"

**Why they work**: High intent — users checking for a specific issue are already aware of the category.

## Tool Planning Checklist

**The tool works if it:**
- [ ] Solves a real problem for the target ICP
- [ ] Produces a result in under 60 seconds with minimal input
- [ ] Has a natural bridge from the result to the product
- [ ] Is shareable — the result is something people want to share or save
- [ ] Can rank for at least one meaningful keyword

**The tool fails if it:**
- [ ] Requires an account to see the result (email capture after value is better)
- [ ] Has too many input fields
- [ ] Produces a generic result that doesn't feel personalized
- [ ] Has no connection to what the product does

---

## Lead Capture Strategy

**Best practice**: Show the result immediately, then offer to email or save it.

Flow:
1. User fills out tool inputs
2. Result displays immediately (never gate the result)
3. Email capture appears below the result: "Email me these results" or "Get a full report"
4. Email capture converts at 20–40% because the user has already seen value

## SEO for Free Tools

- **Target keyword**: "[adjective] [tool type] for [audience/task]" (e.g., "free email subject line tester for marketers")
- **Tool name as keyword**: Make the tool name itself a keyword target
- **Landing page**: The tool should live on its own SEO-optimized page
- **Backlinks**: Free tools earn organic backlinks — promote in newsletters, communities, and directories
